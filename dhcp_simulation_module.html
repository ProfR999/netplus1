How it Works: Untrusted switchports are blocked from forwarding server response payloads (DHCP OFFER / ACK). If a rogue server connected to an untrusted port attempts to spoof, the switch discards the packet immediately.

5. IPAM Address Scope allocation Map (Subnet: 192.168.1.0/24)

Visualizing how an IPAM console maps dynamic pools, exclusion boundaries, and static reservation spaces to avoid IP collision conflicts.

Exclusion (.1 - .30)
Reservations (.31 - .80)
Dynamic Lease Pool (.81 - .220)
Spare (.221 - .254)
Exclusions Range Addresses explicitly blocked from DHCP leasing. Reserved for static deployment on core gateway interfaces, local domain controllers, and server matrices.
MAC Reservations Subnet addresses mapped dynamically but permanently to unique client hardware MACs, guaranteeing printers, servers, and devices always receive the identical IP.
Dynamic Pools Dynamic scopes distributed dynamically with active leases (T1 renewal starts at 50% duration; T2 rebinding starts at 87.5% duration). Unleased addresses return to IPAM console pools.
